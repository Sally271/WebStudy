Git Bash 터미널 계정 변경

1. Windows 자격 증명 변경

Windows에서 '자격 증명' 에 들어가서,
git:https://github.com 에서 '편집'
이후 계정 입력 + 저장 -> 이러면 해당 Repository에 접근할 권한을 획득한 것.

2. 사용자 정보 수정

현재 활성화된 계정 정보 확인을 위해 cmd나 Git Bash에
git config --global user.name 을 치면 확인 가능. 이상한 놈으로 돼있으면
git config --global user.name "userid"
git config --global user.email "useremail" 입력
이후 git config --global user.name 쳐보면 바뀐 것 확인 가능.

의문점

아무 생각 없이 Git Bash 켜서 commit하고 push한건데
왜 다른사람걸로 올라갔지? 거기다가 로그인창까지 떴는데 이러면 내껄로 올라가야하는거 아닌가
거기다가 write 권한 다른사람한테 준적도 없는데;;
